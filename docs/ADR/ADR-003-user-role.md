# ADR-003. User 역할 — Enum 단일 역할

## 상태
확정

## 컨텍스트
User의 역할을 Enum 단일 값으로 관리하거나, Role 엔티티를 별도로 두어 다중 역할을 지원하는 두 가지 방식이 있다.

## 결정
`users.role` 컬럼에 `BOOKER | PROVIDER | ADMIN` Enum으로 단일 역할을 관리한다.

## 근거
- ADR-002에서 역할별 계정 테이블을 분리했다. 다중 역할을 허용하면 한 사람이 `booker_accounts`와 `provider_accounts`를 모두 갖는 케이스가 생기고, 가입/역할 전환 로직이 복잡해진다.
- 이 서비스는 예약자와 제공자가 다른 화면을 쓰는 구조로 설계되었다. 역할이 명확히 분리된 서비스에서 다중 역할은 불필요한 복잡도를 추가한다.
- Spring Security `hasRole()` 처리가 단순해지고, 역할별 분기 코드가 명확해진다.

## 트레이드오프
한 사람이 예약자이면서 제공자가 되려면 별도 계정을 만들어야 한다. 현실적 유연성은 떨어지나, 이 프로젝트 범위에서는 의도적 제약이다.
